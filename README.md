# Лабораторная работа №3 по ООП

## Тема: Иерархия классов (усложнённый вариант)

### Вариант: Менеджер памяти
Разработать приложение, предоставляющее интерфейс для автоматического управления памятью, выделяемой работающим программам. Возможно выделение элемента памяти (1) в виде одиночной переменной, массива и разделяемого несколькими программами сегмента. Возможна также работа со ссылками на существующие переменные этой же программы.

Физически память выделяется менеджером в виде единого для всех приложений массива фиксированных размеров.

Описатель одиночной переменной содержит:

- имя переменной;
- размер выделенной памяти;
- указатель на выделенную память указанного размера.

Описатель массива содержит:

- имя массива;
- размер выделенной памяти;
- размер одного элемента;
- указатель на выделенную память указанного размера.

Описатель разделяемого сегмента памяти содержит:

- имя сегмента;
- размер выделенной памяти;
- размер одного элемента;
- указатель на выделенную память указанного размера;
- список программ, которые могут взаимодействовать посредством сегмента.

Описатель ссылки содержит:

- имя ссылки;
- указатель на другую переменную этой же программы (либо разделяемый сегмент, объявленный в другой программе, к которому имеет доступ текущая программа).

Все данные о выделенной памяти конкретной программы содержатся в некотором контейнере, обеспечивающем удобный доступ к памяти по именам переменных.

Все данные о памяти системы в целом содержатся в таблице (2). Описание каждой зарегистрированной программы содержит, помимо данных о выделенной ей памяти:

- путь к исполнимому файлу;
- квоту памяти (максимальный суммарный объём памяти, который может запросить данная программа).

Возможные случаи некорректного использования памяти должны протоколироваться. При этом выделяются:

- ошибки размеров памяти (доступ за границы массива или разделяемого сегмента);
- утечки памяти (повторное выделение памяти под уже занятый идентификатор без освобождения её или завершение программы до того, как вся запрошенная память освобождена);
- повторное освобождение памяти (попытка освободить уже освобождённую до этого память);
- ошибки доступа по идентификатору (доступ к несуществующей переменной).

Все перечисленные ошибки протоколируются в специальном списке ошибок.

### Обеспечить выполнение следующих операций:

Для переменной:
- запрос значения;
- задание нового значения;
- создание ссылки на переменную с заданным именем.

Для массива и разделяемого сегмента памяти:
- запрос значения элемента массива по индексу;
- запрос значений в диапазоне индексов;
- запись значения в массив.

Для разделяемого сегмента памяти:
- предоставление программе доступа к сегменту;
- отказ программы от работы с сегментом.

Для программы:
- выделить память указанного вида;
- освободить память;
- получить доступ к существующей разделяемой памяти;
- показать всю информацию об используемой памяти;
- показать информацию о разделяемых сегментах памяти, используемых данной программой;
- рассчитать общий объём памяти, используемый данной программой (3).

Для приложения в целом:
- вывести сводку сведений об использовании памяти;
- вывести долю памяти, используемой каждой программой, от суммарного объёма выделенной памяти;
- вывести информацию об ошибках работы с памятью для каждой программы отдельно и для всей системы в целом;
- вывести данные о некорректных ссылках (ссылках на освобождённую память);
- провести дефрагментацию памяти.

### Примечания

(1) Расширяемый класс — элемент памяти. Необходимо предусмотреть возможность переопределения основных операций элемента памяти в новом классе-наследнике (например, сегмент памяти внешнего устройства, в некоторые ячейки которого можно записывать исходные значения для работы устройства, а из других считывать результат работы устройства).

(2) Шаблонный класс — таблица, реализованная сцеплением.

(3) Указанную операцию реализовать в многопоточном режиме. Каждая группа элементов памяти обрабатывается в отдельном потоке.

## Реализация
В данной работе память реализована как вектор фиксированного размера из объектов типа Byte. Эти объекты хранят в себе значение байта (тип ```uint8_t```) и его состояние (выделен ли он: тип ```bool```).
Любая переменная (одиночная, массив или разделяемая переменная) физически представлена как связная область памяти. Она характеризуется адресом первого байта и числом байтов.
Предоставляется возможность глобально управлять разделяемыми элементами, так как они могут существовать независимо от программ. Остальные типы могут существовать только как выделяемые программами.
Информация о программах хранится в глобальной хэш-таблице (класс Table: таблица реализована по аналогии с std::unordered_map).
